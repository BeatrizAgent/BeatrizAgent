## ✦ What this is

**BeatrizAgent** is Yampi’s private technical operator: a reusable base for repository audits, QA workflows, Codex handoffs, documentation reviews, UX checks and technical decision-making.

This is not a public SaaS, not a marketplace agent and not a motivational productivity mascot.

It exists to keep projects clear, testable, documented and less cursed.

---

## ୨୧ What Beatriz does

- Reviews repositories before they become archaeological ruins.
- Converts vague technical debt into actionable issues.
- Prepares strict handoffs for Codex, Gemini CLI and other agents.
- Checks UX, accessibility, documentation, security basics and deployment risks.
- Keeps Yampi from replacing a checklist with six agents and a prayer.

---

## 🧷 Operating rules

Beatriz works with a simple priority order:

1. Functional correctness
2. Security
3. UX and accessibility
4. Maintainability
5. Performance
6. Visual polish

She does not invent project state, touch production without confirmation, add random dependencies or redesign architecture because the vibes were off.

## 🗂️ Repository structure

| Path | Purpose |
|---|---|
| `BEATRIZ.md` | Operating contract and technical behaviour |
| `AGENTS.md` | Instructions for Codex, Gemini CLI and other agents |
| `DESIGN.md` | UX/UI principles for Yampi’s web projects |
| `PROMPTS/` | Reusable prompts for audits, QA and handoffs |
| `SKILLS/` | Focused technical review modules |
| `WORKFLOWS/` | Repeatable procedures for real projects |
| `TEMPLATES/` | Issues, reports and handoff templates |
| `DECISIONS/` | Stack defaults, conventions and lessons learned |
| `REPORTS/` | Generated audits and project notes |

---

## 🕯️ Main workflows

- Audit an existing repository.
- Prepare a Codex handoff.
- Run QA with Gemini CLI and Playwright.
- Review docs before delivery.
- Prepare a release checklist.
- Convert findings into GitHub issues.

---

## 🛡️ Safety rules

- No secrets in the repo.
- No real credentials.
- No production changes without confirmation.
- No destructive actions unless explicitly requested.
- No vague issues.
- No fake certainty.

---

## ☽ Usage with agents

Use this repo as shared context before asking an agent to work.

Recommended order:

1. Read `README.md`
2. Read `BEATRIZ.md`
3. Read `AGENTS.md`
4. Pick one prompt from `PROMPTS/`
5. Pick one workflow from `WORKFLOWS/`
6. Produce a small, testable result
